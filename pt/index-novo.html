5 ★ dos Dados Abertos

Tim Berners-Lee, o inventor da Web e iniciador dos Dados Ligados, sugeriu um esquema de implantação 5 estrelas para Dados Abertos. Aqui, damos exemplos para cada etapa das estrelas e explicamos custos e benefícios que vêm junto com ele.

Exemplos...

Abaixo, apresentamos exemplos para cada nível do plano de Dados Abertos 5 estrelas de Tim. Os dados usados em todos os exemplo são 'a previsão de temperatura para Brasília, Brasil para os próximos 3 dias':

★ torne suas coisas disponíveis na Web (tanto faz o formato) sob uma licença aberta1 exemplo ...
★★ torne-o disponível como dados estruturados (ex. Excel ao invés de imagem escaneada de uma tabela)2 exemplo ...
★★★ utilize formatos não-proprietários (ex. CSV on invés de Excel)3 exemplo ...
★★★★ utilize URIs para identificar coisas, então as pessoas podem apontar para suas coisas 4 exemplo ...
★★★★★ ligue seus dados com dados de outras pessoas para prover contexto5 exemplo ...

Custos & Benefícios ...

Quais são os custos & benefícios de dados Web ★ ?

Como um consumidor ...

  • ✔ Você pode olhar para ele.
  • ✔ Você pode imprimí-lo.
  • ✔ Você pode armazená-lo localmente (no seu disco rígido ou no seu pen drive).
  • ✔ Você pode introduzir os dados em qualquer outro sistema.
  • ✔ Você pode alterar os dados sempre que você quiser.
  • ✔ Você pode compartilhar os dados com quem desejar.

Como um publicador ...

  • ✔ É simples de publicar.
  • ✔ Você não precisa explicar repetidamente aos outros que eles podem utilizar seus dados.
“

É ótimo ter os dados acessíveis na Web sob uma licença aberta (assim como PDDL, ODC-by ou CC0), todavia, os dados estão presos em um documento. Qualquer coisa diferente de escrever um "raspador" customizado, é muito difícil de retirar os dados do documento.

”

Quais são os custos & benefícios de dados Web ★★ ?

Como um consumidor, você pode fazer tudo que pode fazer com dados Web ★ e adicionalmente:

  • ✔ Você pode procesá-los diretamente com um software proprietário para agregá-los, realizar cálculos, visualizá-los, etc.
  • ✔ Você pode exportar isso para outro formato (estrutuardo).

Como um publicador ...

  • ✔ Ainda continua simples para publicar.
“

Esplendido! Os dados estão acessíveis na Web em uma forma estruturada (isto é, legível por máquina), todavia, os dados ainda estão presos em um documento. Para obter os dados você depende de um software proprietário.

”

Quais são os custos & benefícios de dados Web ★★★ ?

Como um consumidor, você pode fazer tudo que pode fazer com dados Web ★★ e adicionalmente:

  • ✔ Você pode manipular os dados de qualquer forma que você queira, sem estar confinado pelas restrições de qualquer software específico.

Como um publicador ...

  • ⚠ Você deve precisar de conversores ou plugins para exportar os dados do formato proprietário.
  • ✔ Ainda é muito simples de publicar.
“

Excelente! Os dados não estão apenas disponíveis ia Web mas agora qualquer um pode utilizar os dados facilmente. Por outro lado, ainda são dados empacotador na Web e não dados na Web.

”

Quais são os custos & benefícios de dados Web ★★★★ ?

Como um consumidor, você pode fazer tudo que pode fazer com dados Web ★★★ e adicionalmente:

  • ✔ Você pode apontar para ele de qualquer outro lugar (na Web ou localmente).
  • ✔ Você pode marcar como favorito.
  • ✔ Você pode reusar parte dos dados.
  • ✔ Você pode reutilizar ferramentas e bibliotecas existentes, mesmo se elas só entenderem partes do padrões que o publicador utilizou.
  • ⚠ Entender a estrutura de um "Grafo" de dados RDF pode ser mis útil do que dados tabulares (Excel/CSV) ou em árvore (XML/JSON).
  • ✔ Você pode combinar os dados com outros dados seguramente. URIs são um esquema global então se duas coisas têm a mesma URI é porque é intencional, e se assim for de certa forma está no caminho para ser dados 5 estrelas!

Como um publicador ...

  • ✔ Você tem controle granular detalhado sobre os itens de dados e pode otimizar seus acessos (balanceamento de carga, cache, etc.)
  • ✔ Outros publicadores de dados podem referenciar seus dados, prmovendo eles a 5 estrelas!
  • ⚠ Você normalmente investe algum tempo separando e fatiando seus dados.
  • ⚠ Você vai precisardefinir URIs pros itens de dados e pensar sobre como representar os dados.
  • ⚠ Você precisa tanto encontrar padrões existentes para reusá-los ou criar os seus próprios.
“

Maravilhoso! Agora são dados na Web. Os (mais importantes) itens de dados possuem uma URI e podem ser compartilhados na Web. Um modo nativo derepresentar dados é utilizando RDF, contudo outros formatos como o Aton podem ser convertidos/mapeados, se necessário.

”

Quais são os custos & benefícios de dados Web ★★★★★ ?

Como um consumidor, você pode fazer tudo que pode fazer com dados Web ★★★★ e adicionalmente:

  • ✔ Você pode descobrir mais dados (relacionados) enquanto consome os dados.
  • ✔ Você pode diretamente aprender sobre o esquema dos dados.
  • ⚠ Agora você precisa lidar com links de dados quebrados, assim como erro 404 nas páginas web.
  • ⚠ Apresentar dados de um link arbitrário como um fato é um risco uma vez que permite pessoas incluírem conteúdo de qualquer página web nas suas páginas. Cuidado, confiança e senso comum são muito ainda necessários.

Como um publicador ...

  • ✔ Você torna seus dados encontráveis.
  • ✔ Você aumenta o valor dos seus dados.
  • ✔ You own organisation will gain the same benefits from the links as the consumers.
  • ⚠ Você vai precisar investir recursos para ligar seus dados a dados de outros na Web.
  • ⚠ Você vai precisar reparar links quebrados ou incorretos.
“

Brilhante! Agora são dados, na Web ligado a outros dados. Ambos consumidor e publicador beneficiam-se pelo efeito rede.

”
